If you have taken a class in statistics, you might have seen the phrase “random variable.” At first, the idea seems pretty intuitive. Many people are already familiar with randomness through everyday examples like rolling dice or flipping coins. So it is tempting to think of a random variable X as something like the result of a die roll.

But that intuition only goes so far. What does it mean to add or subtract an uncertain value? Something feels wrong if you imagine a number that keeps changing every time you try to use it in a calculation. In this article, I want to explain how mathematicians formally define random variables through the example of designing a casino game.

From the Perspective of a Casino Manager

Imagine that you are designing a game at a casino. You want to model a random variable that represents the payout. This makes sense because the payout varies from play to play. But as the casino manager, you want to be clear about the possible outcomes.

Step 1. Define every possible outcome

The game you design is a simple die roll. The player throws a single die and receives a payout based on its outcome. First, you want to define all possible outcomes to which you will assign the payout.

In this case, the relevant outcomes include ⚀, ⚁, ⚂, ⚃, ⚄, ⚅. Later, we will assign probabilities to events built from these outcomes.

This collection of possible outcomes is called the sample space. They are the possible outcomes of the random experiment.

Step 2. Assign probabilities

Now that we know the outcomes of the game, we can assign probabilities to them. If we were working at an honest casino, we could assign fair probabilities to each outcome, namely 1/6 for each face. If not, we could use a weighted die that skews the outcomes. As long as the probabilities are nonnegative and add up to 1, the exact values depend on the model. Mathematically, this corresponds to defining a probability measure.

Step 3. Define the random variable

Once we know the possible outcomes and the events whose probabilities we want to measure, we can define a function that assigns a number to each outcome. This function is called a random variable. Yes, you heard me right: a random variable is a function.

In our casino example, this function assigns a payout to each die roll. For example, we might define X so that odd rolls pay 0 dollars and even rolls pay 2 dollars. The randomness does not come from the function changing its mind. The function is fixed. Instead, the randomness comes from not knowing which die outcome will occur, and the probabilities describe that uncertainty.

What I lied about

Okay, I wish it were this simple. But there is a technical complexity. Here, we only looked at individual outcomes, but surely, we can find the probability of a combination of outcomes. For example, what is the probability of rolling an even number?

So, we need to separate the terminology of random outcomes and random events. A random outcome is a single element in the set of possible outcomes, the sample space. A random event is a subset of the sample space that we agree is measurable, meaning it is the kind of event whose probability we are allowed to assign. In finite examples like a die roll, we usually allow every subset. We call the set of all possible events a σ-algebra (sigma algebra) of the sample space.

Now that I have introduced the concept of random events, we can fix our definitions. When you begin, you have to know not only all possible outcomes but also all possible events. That is because you want to assign probabilities to every possible event, not the individual outcomes. This is easier than what you might expect because once you have well-defined base cases, the rest are almost automatic. For instance, if you know the probability for each face of a die, you can solve for the probability of rolling 1 or 2, or of rolling an even number. In this finite example, once we know the probabilities of the single-face events, we can compute the probabilities of all other events by combining them.

A random variable is not just an ordinary function

If a random variable is a function, what is so special about it? Why do we not call it an ordinary function?

It is because a random variable requires more than just a function. It also needs a sample space, a collection of events, and probabilities assigned to those events. This setup allows us to trace events about the random variable’s output back to events in the sample space, where their probabilities are already defined.

Using the prior example, we have a random variable X that maps even outcomes to 2 and odd outcomes to 0. We will see how it traces events involving X back to events in the original sample space.

Suppose we want to know the probability that X = 2. Then, we can trace back to the outcomes that return 2, which is the set of even outcomes. The set of even outcomes forms a single event in the sample space. So, we can look up the assigned probability for that event. That way, X has a naturally defined probability that X equals 2.

We can extend this to an event involving X. For example, if X ∈ {0,2}, the corresponding event in the sample space is {⚀, ⚂, ⚄} OR {⚁, ⚃, ⚅} (equivalently, {⚀, ⚁, ⚂, ⚃, ⚄, ⚅}). The probability of the entire sample space is 1. So, the event {X = 0 or 2} has a probability of 1.

Important takeaways

A random variable is a function, but it is not just a function. It is a function whose event can be traced back to another event in the sample space, which has an assigned probability. That way, every event involving X has a naturally defined probability.

This perspective also clarifies many expressions involving random variables. For example, a function of random variables, f(X), is merely a composition of functions, like f(g(ω)). If we want to find the probability of an event involving f(X), we trace back through f, then through X, and find the associated event in the sample space. X can be treated just like any other function, but it has an extra ability to trace back and find an associated probability.

So the name “random variable” is slightly misleading. The function X itself is not random; it is a fixed rule. What is random is the input: the outcome from the sample space. Other than that, X is completely determined.

In the formal definition, X is not a variable in the usual sense either. It is a function whose input is often hidden in the background.

Then, why do we call it a random variable in the first place? Well, we use it as if it were random, and we use it as if it were a variable. At the same time, we can do deterministic calculations on it and apply our knowledge of functions to it. The term reflects a compromise: we want to model uncertain quantities, but we also want to calculate with them using the familiar language of functions. So finally, we reached a definition of a random variable that is not random.
